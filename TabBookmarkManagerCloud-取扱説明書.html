Cloud bookmark app / Supabase / PWA

Tab Bookmark Manager Cloud 取扱説明書

PC の Chrome 拡張で保存したタブ情報を、Supabase 経由でスマホや別PCから見られるようにするための単独HTMLアプリです。 Chrome拡張は「現在のタブを高機能に保存する入口」として残し、Cloud版は「複数端末で閲覧・検索・編集する場所」として使います。

保存先Supabase 設定済みなら Cloud、未設定ならブラウザ内の localStorage。
スマホ対応URL、タイトル、タグ、コメントを手入力して保存。ホーム画面追加も可能。
互換性Chrome拡張版の JSON Import / Export 形式をそのまま利用。

最短セットアップ

Supabase テーブルを作成

supabase-schema.sql を Supabase の SQL Editor に貼り付けて実行します。テーブル名は tab_bookmarks です。

Cloud版を開く

output/tab-bookmark-manager-cloud/index.html をブラウザで開きます。GitHub Pages や静的Webサーバーに置いても動きます。

Cloud 設定を保存

ヘッダーの Cloud ボタンから Supabase URL、anon key、共有ID を入力し、「保存して同期」を押します。

同期状態を確認

ヘッダーが sync: 共有ID になれば Cloud 同期が有効です。local の場合はローカル保存です。

推奨: 共有IDは private-tabs-2026 のような推測されにくい文字列にしてください。複数端末で同じ共有IDを使うと同じブックマーク一覧を共有できます。

基本操作

スマホ・Web版で保存

スマホではヘッダーの B! ボタンで保存フォームを開き、URL、タイトル、タグ、コメント を入れて「保存」を押します。URL欄右端の「タイトル取得」でURLからページタイトルを取得できます。タイトルを空にしたまま保存した場合も自動取得を試し、取得できなければURLのドメインから仮タイトルを作ります。

検索

検索欄に語句を入力すると、タイトル、URL、ドメイン、タグ、コメント、ハイライトを対象に絞り込みます。

編集・削除

カードの「編集」からタイトル、URL、タグ、コメントを変更できます。「削除」は確認後に対象カードを削除します。

Import / Export

Chrome拡張版と同じ JSON 配列形式を読み書きします。既存の tab-bookmarks-YYYY-MM-DD.json をCloud版へ移行できます。

URL重複時の動作

Web版で同じURLを保存しようとすると確認が出ます。更新を選ぶと既存カードのタイトル、タグ、コメント、保存日時を新しい入力内容で更新します。

Chrome拡張との連携

Chrome拡張版は、現在のタブからタイトル、URL、選択テキスト、ページ画像、カラーパレット、RSSフィードURLなどを取得できます。 Cloud設定済みの場合、これらの情報を tab_bookmarks に保存します。

同期の考え方

このアプリはログイン認証ではなく、Supabase の anon key と共有IDでデータを分けます。 同じ Supabase URL、anon key、共有IDを設定した端末は、同じ profile_id のデータを読み書きします。

セキュリティ上の注意: anon key と共有IDを知っている人は同じデータにアクセスできます。個人用・少人数用の運用を想定しています。

初回Cloud設定時

既存Cloudデータを取得し、ローカルにあるブックマークとID単位でマージしてからCloudへ保存します。これにより、空のローカルデータでCloud側を消してしまう事故を避けます。

通常保存時

保存、編集、削除、インポート、全削除の後にCloudへ反映します。Cloud保存に失敗した場合も、入力内容はローカルに保持されます。

トラブル対応

local のままで同期されない

Cloud設定が未保存、またはURL・anon key・共有IDのいずれかが空です。Cloudボタンから再入力してください。

Cloud接続に失敗する

supabase-schema.sql を実行済みか、Supabase URLが https://xxxxx.supabase.co 形式か、anon keyを使っているか確認してください。

スマホで保存できない

URL欄に完全なURLを入力してください。例: https://example.com/。example.com だけではURLとして扱われません。

画像が表示されない

初期版は画像ファイル本体を保存せず、画像URLだけを保存します。元サイトが画像直リンクを拒否している場合、サムネイルが表示されないことがあります。

別端末に同じデータが出ない

Supabase URL、anon key、共有IDが完全に同じか確認してください。共有IDが1文字でも違うと別データとして扱われます。

変更時の確認コマンド

node --check output/chrome-extension/popup.js
node --check <(perl -0ne 'while (/<script(?: [^>]*)?>(.*?)<\\/script>/sg) { print $1, "\n" }' output/tab-bookmark-manager-cloud/index.html)

HTML説明書自体は静的HTMLです。見た目を変えた場合は、PC幅とスマホ幅の両方で本文、表、コードブロックが横にはみ出さないことを確認してください。
